Overview

This is UBIF and SDD version 1.1. For more information on SDD see the SDD Wiki, especially the Charter.

Note: While this document is human-readable, RDDL-aware processors may use it to automatically retrieve resources such as different schema versions for validating documents. See http://www.rddl.org for further information.

The root (target) namespace of UBIF and SDD is: http://rs.tdwg.org/UBIF/2006.

  • W3C XML Schema for validation (normative)
    • Note 1: Although RDDL-aware processors may automatically detect a schema location, most processors require an explicit location. SDD instance documents intended for validation should therefore contain the following attributes in the "Datasets" root element:
      xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
      xsi:schemaLocation="http://rs.tdwg.org/UBIF/2006 http://rs.tdwg.org/UBIF/2006/Schema/1.1/SDD.xsd"      .
    • Note 2: Please use a local, cached copy for repeated validations, specifying, e. g.:
      xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
      xsi:schemaLocation="http://rs.tdwg.org/UBIF/2006 http://your-domain.net/path/SDD.xsd"      .
    •
  • W3C XML Schema for validation of single or mixed objects (non-normative)
    • Note 1: This is an alternative primary schema intended for exchange of single UBIF or SDD objects, or a collection of mixed, unrelated objects. When using this schema, objects may contain unresolved references (validation is lower than when using the full document structure). However, this schema is preferable when a client and server exchange objects on specific requests.
    • Note 2: The recommended attributes in the "Datasets" root element for this variant are:
      xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
      xsi:schemaLocation="http://rs.tdwg.org/UBIF/2006 http://rs.tdwg.org/UBIF/2006/Schema/1.1/SDD_ForObjects.xsd"      .
      Again, it is recommended to use a local, cached copy for repeated validations.
